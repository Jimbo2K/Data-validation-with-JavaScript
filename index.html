Functions with JavaScript and regular expressions for data validation (especially for forms)

Although HTML 5 implements its own formulary validation system, its performance depends largely on the browser used. So maybe it won't work in every browser.

Also in some cases it requires specific validations that HTML 5 can not performed (eg: the date format varies by country and it is important to adapt it for improving accessibility).

Before continuing you must notice than data validation, especially for forms, must be twofold, browser's side and especially server's side. The code used in this repository corresponds only to the browser's side.

Using JavaScript

The code you'll find in this repository is just JavaScript, there are no dependencies with other libraries like JQuery or similars.

This feature along with the structure of the code itself, a collection of independent functions, allows it to be easily integrated in other projects.

Using regular expressions

Validations can be performed using only JavaScript code, even if mixed with regular expressions is greatly minimized.

As not everyone is acquainted with using regular expressions, I've tried to make two versions for each validation function, with and without mixture of js and regular expressions.

You can find more about regular expressions in this link from MDN.

Jvalidations code

As it was explained before, JValidations contains JavaScript-only functions, with no external dependencies.
That's why mostly all functions can be "copy & paste" in other code projects, without linking the complete library. However some functions have dependencies with others auxiliar functions inside the library, so if someone wants to copy a function with a dependence, also has to copy the auxiliar functions.

All code is commented (in english and spanish) as one of the objectives of this project is to have a didactic purpose.

JValidations' functions(25/07/2016)

  • ValidFloat1(pnum) Float (positive or negative) number validation

    Arguments: pnum string or number.

    Output: true if pnum is a float number (integer numbers are also considered float), else false.

    Note: no regular expressions used.

  • stringIsFloat(pnum) Float (positive or negative) number validation

    Arguments: pnum string or number.

    Output: true if pnum is a float number (integer numbers are also considered float), else false.

  • ValidPFloat(pnum) Positive float number validation

    Arguments: pnum string or number.

    Output: true if pnum is a positive float number (integer numbers are also considered float), else false.

    Note: no regular expressions used.

  • stringIsPFloat(pnum) Positive float number validation

    Arguments: pnum string or number.

    Output: true si pnum is a positive float number (integer numbers are also considered float), else false.

  • ValidNFloat(pnum) Negative float number validation

    Arguments: pnum string or number.

    Output: true si pnum is a negative float number (integer numbers are also considered float), else false.

    Note: no regular expressions used.

  • stringIsNFloat(pnum) Negative float number validation

    Arguments: pnum string or number.

    Output: true si pnum is a negative float number (integer numbers are also considered float), else false.

  • ValidFloat(pnum,psign) Float (positive or negative), float positive or float negative number validation

    Arguments: pnum string or number;psign(Optional) if 1 it will validate positive float number, for -1 it will validate float negative numbers, for other values it will only check for float number, positive or negative.

    Output: true if pnum is a float number (watch for psign argument), else false.

  • ValidInt1(pnum) Integer (positive or negative) number validation

    Arguments: pnum string or number.

    Output: true if pnum is an integer, else false.

    Note: no regular expressions used.

  • stringIsInt(pnum) Integer (positive or negative) number validation

    Arguments: pnum string or number.

    Output: true if pnum is an integer, else false.

  • ValidPInt(pnum) Integer positive number validation

    Arguments: pnum string or number.

    Output: true if pnum is a positive integer, else false.

    Note: no regular expressions used.

  • stringIsPInt(pnum) Integer positive number validation

    Arguments: pnum string or number.

    Output: true if pnum is a positive integer, else false.

  • ValidNInt(pnum) Integer negative number validation

    Arguments: pnum string or number.

    Output: true if pnum is a negative integer, else false.

    Note: no regular expressions used.

  • stringIsNInt(pnum) Integer negative number validation

    Arguments: pnum string or number.

    Output: true if pnum is a negative integer, else false.

  • ValidInt(pnum,psign) Integer (positive or negative), integer positive o integer negative number validation

    Arguments: pnum string or number;psign(Optional) if 1 it will validate positive integer number, for -1 it will validate negative integer number, for other values it will only check for integer number, positive or negative.

    Output: true si pnum is an integer number (watch for psign argument), else false.

  • stringIsHex(pnum) Hexadecimal number validation (also for hexadecimal color format in CSS).

    Arguments: pnum string.

    Output: true if pnum is an hexadecimal number or a valid hexadecimal CSS color (#xyz or #xxyyzz), else false.

  • codeIsbn10(pisbn) 10 digits ISBN code validation

    Arguments: pnum string or number.

    Output: true if pisbn is a valid 10 digits ISBN (10th must be the valid control code, calculated using the other 9 digits).

  • codeIsbn13(pisbn) 13 digits ISBN code validation

    Arguments: pnum string or number.

    Output: true if pisbn is a valid 13 digits ISBN (13th must be the valid control code, calculated using the other 12 digits).

  • codeIsbn(pisbn) ISBN code validation, for 10 and 13 digits format

    Arguments: pnum string or number.

    Output: true if pisbn is a valid ISBN code (the last digit must be a valid control code, calculated using the other digits).

    Dependencies: codeIsbn10() and codeIsbn13().

  • stringIsYear(pyear) String as a year with A.D. format validation (nor the Date object format)

    Arguments: pyear string.

    Output: true if pyear is a year with A.D. format ("1345", "1345 A.D.", "700 B.C.").

  • stringIsDate(pyear,page) String as a date validation

    Arguments: pyear string as dd<sep>mm<sep>aaaa where <sep>=' ' , '_' , '-' , '/' , '\' , '.'; page(Optional) a boolean, if true the date will be considered birth-date.

    Output: array whose first index will be true or false if the date is valid or not. If false the next four index will be error messages to identify the possible causes.

    Dependencies: findSep().

  • fixedPhone(ptel,pcountry) String as a fixed phone number validation for several european countries (Sapin, France, Ireland and Italy for the moment). If the country is not indicated it will take the indicated by the browser. If the country is not available the default country is Spain.

    Arguments: ptel string; pcountry(Optional) an string with the country code (country codes).

    Output: true if ptel is a valid fixed phone number, with the format for the country selected.

  • stringIsName(pname) A name (first or last name) can not contain numbers or symbols

    Arguments: pname string.

    Output: true if pname is not an empty string and doesn't contain numbers or symbols.

  • stringIsEmail(pemail) String with e-mail format validation

    Arguments: pemail string.

    Output: true if pemail has a valid format for an e-mail (the function doesn't check if it is a real e-mail).

  • findSep(pstring) Looks for separation symbols or gaps in a string (' ' , '-' , '/' , '\' , '_') and returns the first coincidence (it assumes that only one type of separator is used)

    Arguments: pstring string.

    Output: string with the separation symbol (e.g. for "28/02/2013" the returned string will be "/").

  • onlyText(pstring) Removes from a string any character other than a letter or a blank. "ñ", "Ñ", "ü", "Ü" and spanish accent marks are considered letters.

    Arguments: pstring string.

    Output: string is the same as pstring but with no numbers and symbols.

Funciones con js y expresiones regulares para validación de datos (especialmente formularios)

Aunque HTML 5 implementa su propio sistema de validación para formularios, su funcionamiento depende en gran medida del navegador que utilice el formulario pudiendo funcionar o no.

Además en algunos casos será preciso realizar validaciones específicas que HTML 5 no puede llevar a cabo (por ejemplo: el formato de fecha varía según el país y es importante adaptarlo por accesibilidad).

Antes de contniuar debes tener en cuenta que la validación de datos, especialmente para formularios, debe ser doble, del lado del navegador y especialmente en el servidor. El código de este repositorio corresponde sólo al lado del navegador.

Uso de JavaScript

El código que encontrarás en este repositorio es sólo JavaScript, sin dependencias con otras librerias como JQuery o similares.

Esta característica junto a la estructura del propio código, una colección de funciones independientes, permite que pueda ser integrado de manera sencilla en otros proyectos.

Uso de expresiones regulares

Las validaciones se pueden realizar empleando sólo código JavaScript, aunque si se mezcla con expresiones regulares se puede minimizar en gran medida.

Como no todo el mundo está familiarizado con el uso de expresiones regulares, he intentado hacer dos versiones para cada función de validación, con y sin mezcla de expresiones regulares y js.

You can find more about regular expressions in this link from MDN.

El código de Jvalidations

Como ya se ha explicado anteriormente la librería JValidations contiene una serie de funciones escritas por completo en Javascript, sin ninguna dependencia con librerías externas.
Por esta razón la mayoría de funciones se pueden "copiar y pegar" en otros proyectos de código, sin necesidad de enlazar la librería completa. Sin embargo algunas funciones tienen dependencias con otras auxiliares del mismo código, de manera que si se quieren copiar también se deberán copiar las auxiliares.

Todo el código está comentado (en español e inglés) ya que uno de los objetivos de este proyecto es que tenga una finalidad didáctica.

Funciones integradas en JValidations (25/07/2016)

  • ValidFloat1(pnum) Validación de número real, positivo o negativo

    Argumentos: pnum string o número.

    Salida: true si pnum es un nro. real (los enteros también se consideran reales), false si no lo es.

    Nota: sin expresiones regulares.

  • stringIsFloat(pnum) Validación de número real, positivo o negativo

    Argumentos: pnum string o número.

    Salida: true si pnum es un nro. real (los enteros también se consideran reales), false si no lo es.

  • ValidPFloat(pnum) Validación de número real positivo

    Argumentos: pnum string o número.

    Salida: true si pnum es un nro. real positivo (los enteros también se consideran reales), false si no es un número o un número positivo.

    Nota: sin expresiones regulares.

  • stringIsPFloat(pnum) Validación de número real positivo

    Argumentos: pnum string o número.

    Salida: true si pnum es un nro. real positivo (los enteros también se consideran reales), false si no es un número o un número positivo.

  • ValidNFloat(pnum) Validación de número real negativo

    Argumentos: pnum string o número.

    Salida: true si pnum es un nro. real negativo (los enteros también se consideran reales), false si no es un número o un número negativo.

    Nota: sin expresiones regulares.

  • stringIsNFloat(pnum) Validación de número real negativo

    Argumentos: pnum string o número.

    Salida: true si pnum es un nro. real negativo (los enteros también se consideran reales), false si no es un número o un número negativo.

  • ValidFloat(pnum,psign) Validación de número real positivo, neagtivo o indistintamente de su signo

    Argumentos: pnum string o número;psign(Opcional) si es 1 validación de reales positivos, para -1 validación de reales negativos, si no se utiliza o se pasa un valor distinto de los anteriores la validación es para reales positivos o negativos.

    Salida: true si pnum es un nro. real válido (ver psign), false si no es válido.

  • ValidInt1(pnum) Validación de número entero, positivo o negativo

    Argumentos: pnum string o número.

    Salida: true si pnum es un nro. entero, false si no lo es.

    Nota: sin expresiones regulares.

  • stringIsInt(pnum) Validación de número entero, positivo o negativo

    Argumentos: pnum string o número.

    Salida: true si pnum es un nro. entero, false si no lo es.

  • ValidPInt(pnum) Validación de número entero positivo

    Argumentos: pnum string o número.

    Salida: true si pnum es un nro. entero positivo, false si no es un número o un número entero positivo.

    Nota: sin expresiones regulares.

  • stringIsPInt(pnum) Validación de número entero positivo

    Argumentos: pnum string o número.

    Salida: true si pnum es un nro. entero positivo (los enteros también se consideran reales), false si no es un número o un número positivo.

  • ValidNInt(pnum) Validación de número entero negativo

    Argumentos: pnum string o número.

    Salida: true si pnum es un nro. entero negativo (los enteros también se consideran reales), false si no es un número o un número negativo.

    Nota: sin expresiones regulares.

  • stringIsNInt(pnum) Validación de número entero negativo

    Argumentos: pnum string o número.

    Salida: true si pnum es un nro. entero negativo, false si no es un número o un número negativo.

  • ValidInt(pnum,psign) Validación de número entero positivo, neagtivo o indistintamente de su signo

    Argumentos: pnum string o número;psign(Opcional) si es 1 validación de reales positivos, para -1 validación de reales negativos, si no se utiliza o se pasa un valor distinto de los anteriores la validación es para reales positivos o negativos.

    Salida: true si pnum es un nro. entero válido (ver psign), false si no es válido.

  • stringIsHex(pnum) Validación de numero hexadecimal (también funciona con colores CSS en formato hexadecimal).

    Argumentos: pnum string.

    Salida: true si pnum es un número hexadecimal o unc color CSS en formato hexadecimal (#xyz o #xxyyzz), en caso contrario false.

  • codeIsbn10(pisbn) Validación de código ISBN de 10 dígitos

    Argumentos: pnum string o número.

    Salida: true si pisbn es un código ISBN de 10 dígitos válido (el 10º dígito corresponde al código de control calculado en base a los 9 anteriores).

  • codeIsbn13(pisbn) Validación de código ISBN de 13 dígitos

    Argumentos: pnum string o número.

    Salida: true si pisbn es un código ISBN de 13 dígitos válido (el 13º dígito corresponde al código de control calculado en base a los 12 anteriores).

  • codeIsbn(pisbn) Validación de código ISBN de 10 ó 13 dígitos

    Argumentos: pnum string o número.

    Salida: true si pisbn es un código ISBN válido (el último dígito corresponde al código de control calculado en base a los anteriores).

    Dependencias: codeIsbn10() y codeIsbn13().

  • stringIsYear(pyear) Validación de una cadena como año en formato 'A.D.' (no el formato del Objeto Date Js)

    Argumentos: pyear string.

    Salida: true si pyear es un año con formato válido (p.ej. "1345", "1345 A.D.", "700 B.C.").

  • stringIsDate(pyear,page) Validación de una cadena con distintos formatos de fecha posible

    Argumentos: pyear string con formato dd<sep>mm<sep>aaaa y <sep>=' ' , '_' , '-' , '/' , '\' , '.'; page(Opcional) un booleano que si vale true la fecha se considera de nacimiento.

    Salida: array cuyo primer elemento será true o false si la fecha es o no válida. En caso de ser no válida los siguientes 4 elementos contendrán mensajes para identificar el fallo.

    Dependencias: findSep().

  • fixedPhone(ptel,pcountry) Validación de una cadena como número de teléfono fijo de distintos paises europeos (actualmente España, Francia, Irlanda e Italia) Si no se indica el país chequeará el lenguaje del navegador para asignarlo. Si no estuviese disponible tomará por defecto España.

    Argumentos: ptel string; pcountry(Opcional) un string con el código del país para chequear el formato (códigos de paises).

    Salida: true si ptel es un teléfono fijo con formato válido para el país indicado.

  • stringIsName(pname) Un nombre (de pila o/y apellidos) no puede contener números o símbolos

    Argumentos: pname string.

    Salida: true si pname no es una cadena vacía, no contiene ni números ni símbolos.

  • stringIsName(pemail) Valida si el formato de una cadena se corresponde con el de un e-mail

    Argumentos: pemail string.

    Salida: true si pemail tiene el formato de un e-mail (no chequea si es un e-mail real).

  • findSep(pstring) Busca en una cadena símbolos de separación (' ' , '-' , '/' , '\' , '_') y devuelve el primero que encuentra (se presupone que se utiliza sólo un tipo)

    Argumentos: pstring string.

    Salida: string con el símbolo que ha encontrado (p. ej. para "28/02/2013" la salida será "/").

  • onlyText(pstring) Elimina de una cadena cualquier carácter que no sea una letra o un espacio en blanco. Admite "ñ", "Ñ", "ü", "Ü" y acentos españoles.

    Argumentos: pstring string.

    Salida: string igual a pstring pero con números y símbolos eliminados.
